Web Accessibility

Web accessibility means that websites, tools, and technologies are designed and developed so that people with disabilities can use them.

Assistive devices for web accessibility include:

  • Head pointers
  • Motion tracking or eye tracking
  • Large-print and tactile keyboards

Semantic Tags

header tag

The header element represents a container for introductory content or a set of navigational links. It typically contains

  • one or more heading elements
  • logo or icon
  • authorship information

    • footer

    The footer tag defines a footer for a document or section. A footer element typically contains:

  • authorship information
  • copyright information
  • contact information
  • sitemap
  • back to top links
  • related documents

    • nav

    The nav tag defines a set of navigation links. Notice that NOT all links of a document should be inside a nav element. The nav element is intended only for major blocks of navigation links. Browsers, such as screen readers for disabled users, can use this element to determine whether to omit the initial rendering of this content.

    article

    An article should make sense on its own and it should be possible to distribute it independently from the rest of the site.

    aside

    The aside content is often placed as a sidebar in a document.

    Benefits of using semantics tags

    • It has greater accessibility.
    • It helps search engines and other user devices to determine the importance and context of web pages.
    • Pages made with semantic elements are much easier to read.
    • It offers a better user experience.
    • It's easier for computers to understand what content is in your document and how it is structured.
    • Screen readers will also have more information about what is on the page.

    What is HTML

    HTML stands for Hyper Text Markup Language. It is used to design web pages using a markup language. HTML is an abbreviation of Hypertext and Markup language. HTML 5 is the fifth and current version of HTML. It has improved the markup available for documents and has introduced application programming interfaces (API) and Document Object Model (DOM).
